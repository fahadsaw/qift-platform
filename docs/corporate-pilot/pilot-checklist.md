# Pilot Campaign Checklist (per campaign)

## D-10 — Lock
- [ ] Prepayment received and reconciled
- [ ] Campaign spec locked (gift, groups, copy, schedule, claim window)
- [ ] Support line staffed + scripts ready (AR-first)

## D-7 — Roster + capacity
- [ ] Roster validated; company signed off final list
- [ ] Any uploaded address data dropped + company notified
- [ ] Merchant capacity confirmation in writing (units, window, cities)
- [ ] Coverage sanity: gift is courier-able OR coverage acknowledged + fallback agreed

## D-3 — Rehearsal
- [ ] 5 internal test numbers through the FULL flow (SMS → claim → address)
- [ ] Claim page shows NO name/company before OTP (the F1 check — verify every time)
- [ ] Opt-out path tested
- [ ] Kill-switch decision owner named for dispatch day

## D0 — Dispatch
- [ ] Waves ≤250/hour; pause point between waves
- [ ] Sender ID correct; support contact in message
- [ ] Live monitoring: bounce rate alarm (>10% → PAUSE + verify roster)

## D+1 → D+7 — Claim window
- [ ] Addresses entered ONLY into the vault (two named people)
- [ ] D+2 automatic re-send to non-responders (one, polite)
- [ ] "This isn't me" reports → void token, flag row, notify company (no details)
- [ ] D+5 unreached list (names + status only) → company for internal nudge

## D+7 — Handoff
- [ ] Claim window closed; per-merchant fulfillment lists cut (name, address, item, window — nothing else)
- [ ] Secure handoff; merchant confirms receipt + delete-after-delivery obligation

## D+7 → D+14 — Delivery
- [ ] Daily merchant status updates reconciled
- [ ] Exceptions worked recipient-direct (never via the company)
- [ ] Replacements: substitute only per pre-approved ladder + notify recipient

## D+14 — Close
- [ ] Completion report (funnel numbers, zero PII beyond names + status)
- [ ] Final invoice reconciliation (gift cost on claimed; fee per contract)
- [ ] Retro interview (45 min) — friction log entries filed
- [ ] Case-study assets collected per contract clause

## D+30 — Purge
- [ ] Roster + addresses purged from vault; merchant deletion confirmed
- [ ] Purge certificate issued to company
- [ ] Pilot metrics filed: claim %, delivery %, tickets, NPS-ish quote

**Gate G1 inputs:** claim ≥80% in window · delivery ≥95% of claimed ·
invoice paid undisputed · "would you run Eid through this?" = yes.
